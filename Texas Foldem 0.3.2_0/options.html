It's not working?

You might have to still read the important information below (don't skip it) and check the checkbox acknowledging you've read it and agree.

If that's not the issue, hit F-12 (or right-click the page and hit inspect then navigate to console tab) and see if the console has any red errors talking about CORS issues. If this is the case install this extension and then click on the extension to enable it

Another reason it might not be working is because you were not seated at the table yet, or the game wasn't started when you clicked 'run'. Just hit run again.

"Bugs"

  • Sometimes when we call a single-raised pot from BB, because of the pot-size mutation observer, we will sometimes hit fold as soon as we call as it considers our own call as a 3bet. It's on the to-dos to fix this. But usually this doesn't matter because either A. We are closing the action so nothing happens (and would fold our non-3bettable hand to a backraise anyway). This could become an issue if we flat 76s and then one of the limpers decides to min-3bet and everyone else calls pricing us in. If this happens we can react to the fold-button turning red anyway so it's a low-priority bug

  • UTG will always have the beeping noise because the script takes between a third of a second and 1.5 seconds (randomized) to fold each hand [won't be fixing this btw]

  • Playing with straddles still occasionally will cause silent errors but the worst case scenario is that a hand that should be auto-folded preflop ends up not being folded

  • This isn't really even a bug but the run button will still show the activation color even if it fails to run.. just something to note

Future features

  • You can start the extension while the game is idle/not started yet and regardless if you are seated or not. This usually makes an error because the script needs to detect the dealer-button on the table; which is why the script will always fail when there is a dead button since it's classified as a different element. I could debug this but it could take me 2ish hours or so and it's such a rare edge-case - you can just fold that single hand on your own.

  • A little checkbox on the bottom left of the screen above the log/ledger button which allows you to automatically stand before you post your next BB

  • Be able to be more selective with continuing hands facing 3bets/4bets. For now all pocket-pairs will not auto-fold facing 3bets.
    We also need to be more distinctive of whether or not the 3bet we are facing with pocket 6s is cold or is it against our 2bet?

  • Position-dependant continuing ranges
    Ex: You might defend BB with A6o vs 4bb CO open but fold it to EP open. You might not fold AJo at SB vs HJ open but might want to fold it vs EP open

  • Being able to set custom ranges for each of the position v position situations. Maybe with some range chart drop-down. Each user of the extension will need a way to save their custom ranges which makes this much more complicated than just having my own hard-coded ranges.

  • ***Stretch goal***: if enough people are using this and willing to pay for this feature: Pot odds considerations so the decision to fold or not (preflop obviously) will be more dynamic by the bot but still be the same decision you would come to on your own.

    Example: you have 66 at BB and a loose player who never 4bets (bot doesn't know this data-point) opens UTG, player after him 3bets to 3x, and then 3 other players flat the 3bet before you. You are getting pot odds to call here and folding would be too nitty. We want to not fold so tight and give the user the chance to decide if this flat is worth it or not depending on how TAG they think UTG is

    Example2: You have k9s at the HJ 10 handed and have your ranges set to a nitty configuration that always folds this. You may elect to flat or limp along this hand if enough players behind also call. Using pot odds consideration would un-click (or re-click) fold button once the threshold to limp-along/flat has been met
